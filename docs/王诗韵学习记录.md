## 第 1 周个人学习记录

**姓名**：王诗韵\

**本周完成代码文件**：`consistency/event_schema.py`、`dashboard/app.py`

### 1. 本周学习的理论

- **冲突一致性**：多个边缘节点对同一设备的状态判断可能产生冲突，需要统一事件模型来描述和处理这些冲突
- **事件驱动架构**：通过定义标准化的事件对象，实现云边之间的结构化通信和状态传递
- **可视化仪表盘**：将系统运行状态、决策过程和关键指标以直观的方式展示

### 2. 理论如何落到代码里

- **冲突一致性**：通过 `event_schema.py` 中的 `Event` 类定义统一的事件格式，使得多个边缘节点的决策结果可以被统一接收和比较，为后续冲突检测和仲裁打下基础。
- **事件驱动架构**：`Event` 类的序列化/反序列化方法（`to_dict()` 和 `from_dict()`）确保了云边之间可以通过 JSON 格式交换事件数据，字段验证机制保证了数据的一致性和正确性。
- **可视化仪表盘**：`dashboard/app.py` 设计了七大模块的布局，覆盖从数据输入到最终决策的完整流程，为后续接入真实数据展示系统运行状态做好准备。

### 3. 遇到的问题

- Streamlit 页面布局调整需要反复调试，特别是网格布局和动态效果的实现
- Event 类字段设计需要考虑云边协同的多种场景，字段定义需要兼顾完整性和简洁性

### 4. 下周计划

- 完成 `conflict_detector.py`：实现重复告警检测、风险等级冲突检测、动作冲突检测
- 完成 `conflict_resolver.py`：实现高风险优先、高置信度优先、云端优先三种仲裁策略
- Dashboard 接入真实数据，展示冲突检测和仲裁结果
