# 云边智控：面向工业与能源场景的云边协同感知与决策系统

## 项目范围说明

- 文档状态：范围冻结版
- 冻结日期：2026 年 7 月 22 日
- 提交截止时间：2026 年 8 月 31 日 23:59
- 实施原则：双场景共用一套云边协同底座，先完成可运行闭环，再增加算法创新

## 1. 项目目标

面向工业设备预测性维护和分布式能源稳定控制两类差异明显的场景，构建包含分布式感知、边缘轻量推理、风险与时延感知路由、云端复杂复核、多节点冲突仲裁、联邦持续更新和可视化展示的完整系统。

系统需要证明以下价值：

1. 边缘侧能够在毫秒级完成基础感知与决策。
2. 网络正常时，低置信度或高风险任务可以获得云端复核。
3. 弱网或断网时，边缘节点仍能执行安全的基础业务。
4. 多个边缘节点发生关联决策冲突时，系统能够形成满足全局约束的最终动作。
5. 不上传原始数据，通过 Flower 聚合边缘模型参数。

## 2. 场景范围

### 2.1 工业场景：AI4I 设备预测性维护

**数据来源**

- AI4I 2020 Predictive Maintenance Dataset。
- 已划分为 5 个非 IID 工业边缘客户端。
- 公共测试集与客户端训练数据严格隔离。

**边缘输入**

- 空气温度。
- 过程温度。
- 转速。
- 扭矩。
- 刀具磨损。
- 产品类型。

**边缘输出**

- 故障概率。
- 故障类型或正常状态。
- 风险等级。
- 建议动作。
- 校准后的置信度。
- 推理时延和模型版本。

**云端职责**

- 复核低置信度、高风险和分布外样本。
- 综合多个设备事件安排维护优先级。
- 记录困难样本，支持后续模型更新。
- 对边缘决策提供可追溯的复核理由。

**控制动作**

- monitor：继续监测。
- maintain：安排维护。
- reduce_load：降低负载。
- shutdown：安全停机。

### 2.2 能源场景：分布式电网稳定性与协同控制

**数据来源**

- 优先使用 UCI Electrical Grid Stability Simulated Data。
- 数据描述一个发电节点和三个消费节点构成的去中心化四节点电网。
- 任务包括电网稳定性分类以及稳定风险评估。

**边缘输入**

- 节点反应时间。
- 节点发电或用电功率。
- 电价弹性或控制系数。
- 节点状态时间戳。
- 本地网络状态。

**边缘输出**

- 电网稳定概率。
- 本地风险等级。
- 本地调节建议。
- 功率调整请求。
- 置信度和数据新鲜度。

**云端职责**

- 汇总发电节点和消费节点的局部建议。
- 检查供需平衡和全局安全约束。
- 解决多个节点同时请求有限资源的冲突。
- 生成最终全局控制动作。

**控制动作**

- maintain：保持当前状态。
- reduce_load：降低负荷。
- increase_generation：增加发电或供能。
- emergency_limit：进入紧急限载。

## 3. 核心技术主线

系统主线固定为：

分布式场景数据 → 边缘感知模型 → 边缘结构化决策 → 置信度与风险评估 → 动态任务路由 → 云端复核 → 多节点冲突仲裁 → 最终动作 → 日志与可视化 → 联邦更新

核心创新固定为：

1. **风险—时延双约束动态路由**：综合模型不确定性、业务风险、网络 RTT、丢包率、队列长度和 deadline 选择推理路径。
2. **弱网安全自治**：云端超时或断网时执行场景相关的保守策略，并在网络恢复后同步积压事件。
3. **安全一致性仲裁**：使用风险偏序、置信度、数据新鲜度和全局资源约束处理多节点冲突。

## 5. 明确不做的内容

- 不将随机 mock 的结果用于最终效果结论。
- 不将 JSON 合法率等同于模型决策能力。
- 不以训练或部署超大模型作为比赛交付前提。
- 不在主链路未完成时投入 GCM 个性化聚合研究。
- 不为了展示效果伪造网络、时延或准确率数据。
- 不在 2026 年 8 月 28 日后增加新功能。

## 6. 主要技术栈

| 层级 | 选择 |
|---|---|
| 数据与实验 | Python、Pandas、NumPy、scikit-learn |
| 边缘模型 | Logistic Regression、Random Forest 或 MLP；Qwen3-0.6B 为增强模块 |
| 边缘与云端服务 | FastAPI、Pydantic、HTTP API |
| 边缘大模型运行 | llama.cpp 或现有推理服务 |

## 7. 验收目标

### 7.1 赛题硬指标

- 边缘轻量大模型任务能力保持率达到赛题要求的 80%—90%，并明确参考模型和测试集。
- TTFT 相对云端或全量模型基线减少至少 75%。
- 边缘单次推理峰值内存不超过 1.5GB。
- 弱网期间基本业务功能保持率不低于 90%。
- 工业与能源两个场景平均端到端时延低于 0.2 秒。
- 正常工作负载下最终未解决冲突比例不超过 5%。
- 故障注入测试的冲突解决成功率不低于 90%。

### 7.2 工程验收

- 两个场景均可使用一条命令完成训练和评估。
- 两个场景均可通过 HTTP 返回统一结构化决策。
- 每个请求具有 trace_id，可以追踪边缘、路由、云端和仲裁阶段。
- 所有最终报告数字均能追溯到原始结果文件。
- API Key、模型密钥和个人信息不进入仓库。
- 在干净环境中按照 README 可以启动和演示系统。
